# 07 - 订单管理 + 状态机

> 本教程讲解订单模块及状态机设计

## 一、订单模块概述

### 1.1 订单状态流转

```
用户下单 ──▶ 订单提交 ──▶ 商家接单 ──▶ 派送中 ──▶ 完成
   │           │            │           │
   │           │            │           │
   ▼           ▼            ▼           ▼
  待支付    待接单(支付)  已接单     已完成
   │           │            │
   ▼           ▼            ▼
  已取消    已取消(拒单)   已取消
```

### 1.2 订单状态表

```java
// 订单状态常量
public class OrderStatus {
    public static final Integer PENDING_PAYMENT = 1;  // 待支付
    public static final Integer PENDING_RECEIPT = 2; // 待接单
    public static final Integer RECEIVED = 3;         // 已接单
    public static final Integer DELIVERY = 4;        // 派送中
    public static final Integer COMPLETED = 5;        // 已完成
    public static final Integer CANCELLED = 6;       // 已取消
}
```

**Service 实现**:
```java
public OrderSubmitVO submitOrder(OrdersSubmitDTO dto) {
    // 1. 获取购物车数据
    List<ShoppingCart> carts = shoppingCartMapper.list(userId);
    
    if (carts == null || carts.isEmpty()) {
        throw new BusinessException("购物车为空");
    }
    
    // 2. 创建订单
    Orders order = new Orders();
    order.setNumber(String.valueOf(System.currentTimeMillis())); // 订单号
    order.setStatus(OrderStatus.PENDING_PAYMENT);
    order.setUserId(userId);
    order.setAddressBookId(dto.getAddressBookId());
    order.setOrderTime(LocalDateTime.now());
    order.setAmount(calculateAmount(carts)); // 计算总金额
    
    // 3. 保存订单
    orderMapper.insert(order);
    
    // 4. 保存订单明细
    carts.forEach(cart -> {
        OrderDetail detail = new OrderDetail();
        detail.setOrderId(order.getId());
        detail.setName(cart.getName());
        detail.setImage(cart.getImage());
        detail.setNumber(cart.getNumber());
        detail.setAmount(cart.getAmount());
        detailMapper.insert(detail);
    });
    
    // 5. 清空购物车
    shoppingCartMapper.deleteByUserId(userId);
    
    return new OrderSubmitVO(order.getId(), order.getNumber(), order.getAmount());
}
```

## 四、订单状态机

### 4.1 状态流转图

```
┌──────────┐  支付    ┌──────────┐  接单    ┌──────────┐  派送    ┌──────────┐
│  待支付   │ ──────▶ │  待接单   │ ──────▶ │  已接单   │ ──────▶ │  派送中  │
└──────────┘          └──────────┘          └──────────┘          └──────────┘
     │                    │                    │                    │
     │                    │                    │                    │
     ▼                    ▼                    ▼                    ▼
┌──────────┐         ┌──────────┐          ┌──────────┐         ┌──────────┐
│  已取消   │         │  已取消   │          │  已取消   │         │  已完成   │
└──────────┘         └──────────┘          └──────────┘         └──────────┘
```

### 4.2 状态校验

```java
// 只有待接单状态可以接单
public void confirm(Long orderId) {
    Orders order = orderMapper.getById(orderId);
    if (order.getStatus() != OrderStatus.PENDING_RECEIPT) {
        throw new BusinessException("订单状态错误");
    }
    order.setStatus(OrderStatus.RECEIVED);
    orderMapper.update(order);
}

// 只有派送中状态可以完成
public void complete(Long orderId) {
    Orders order = orderMapper.getById(orderId);
    if (order.getStatus() != OrderStatus.DELIVERY) {
        throw new BusinessException("订单状态错误");
    }
    order.setStatus(OrderStatus.COMPLETED);
    orderMapper.update(order);
}
```

## 七、常见问题

### Q1: 订单超时未支付？
使用定时任务扫描超时订单并自动取消

### Q2: 订单如何防止重复提交？

### Q3: 如何实现乐观锁？
