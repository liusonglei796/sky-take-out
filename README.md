# 苍穹外卖 (Sky Take Out)

## 📌 项目简介

“苍穹外卖”是一个专门为餐饮企业（餐厅、饭店）定制的一款软件产品。该系统包含了**系统管理后台**和**移动端应用**两部分。
- **系统管理后台**：主要提供给餐饮企业内部员工（如管理员、店长、普通员工）使用，可以对餐厅的分类、菜品、套餐、订单、员工、数据统计等情况进行统一的管理维护。
- **移动端微信小程序**：主要提供给消费者使用，可以在线浏览菜品、添加购物车、下单、支付、催单等。

## 🏗️ 模块说明

本项目采用 Maven 多模块化构建，主要包含以下子模块：

- `sky-common`: **公共模块**，存放工具类、常量类、异常类集、配置类等，提供各模块通用的基础能力。
- `sky-pojo`: **实体类模块**，存放 Entity（数据库实体）、DTO（数据传输对象）、VO（视图对象）等。
- `sky-server`: **后端服务模块**，整个项目的核心业务代码，包含了 Controller 层、Service 层、Mapper 层的实现，以及各类拦截器和配置文件。

## 🛠️ 技术选型

### 后端技术
- **核心框架**：Spring Boot 2.7.3
- **持久层框架**：MyBatis
- **数据库**：MySQL
- **缓存**：Redis
- **接口文档**：Knife4j (Swagger 增强版)
- **分页插件**：PageHelper
- **连接池**：Druid
- **对象存储**：阿里云 OSS
- **鉴权认证**：JWT
- **其他**：Lombok、Apache POI、WeChat Pay (微信支付)

## 📦 环境要求

在运行本项目前，请确保您的本地环境中已安装并配置好以下组件：

- **JDK**: 1.8+
- **Maven**: 3.6+
- **MySQL**: 5.7+ 或 8.0+
- **Redis**: 5.0+

## 🚀 快速上手

### 1. 准备数据库
1. 在本地 MySQL 数据库中创建相应的数据库（如 `sky_take_out`）。
2. 将项目提供的 SQL 脚本导入到该数据库中，完成表结构的初始化。

### 2. 配置修改
在 `sky-server/src/main/resources` 目录下找到 `application.yml` 和 `application-dev.yml`（或相应的环境配置文件），并修改以下关键配置：
- **数据源配置**：修改 `spring.datasource.url`、`username` 和 `password` 以指向您的本地 MySQL。
- **Redis配置**：修改 `spring.redis.host`、`port` 及 `password`（如有）以连接本地 Redis。
- **阿里云OSS与微信支付 （可选/测试阶段）**：如果需要图片上传或支付功能，请修改配置中的相应密钥信息 （也可以根据需要将其注释或 mock 相关业务实现）。

### 3. 编译并运行
1. 在项目根目录下，执行 `mvn clean install` 进行编译和构建。
2. 找到 `sky-server` 模块下的启动类 `SkyApplication`，直接运行即可。
3. 系统启动成功后，默认可在浏览器访问接口文档排查功能：
   - 接口文档地址（Knife4j）：`http://localhost:8080/doc.html`

## 💡 核心功能概览

### 管理端后台
- **员工管理**：员工的增删改查、启用/禁用。
- **分类管理**：菜品分类和套餐分类的管理。
- **菜品管理**：菜品的新增、修改、状态起售/停售、图片上传等。
- **套餐管理**：包含多道菜品的组合套餐的维护。
- **订单管理**：查看用户订单，并进行接单、拒单、派送、完成等操作。
- **数据统计**：基于 Apache POI 的图形化数据报表（营业额、用户统计、订单统计、销量排名等）。

### 用户端小程序
- **点餐浏览**：根据分类展示菜品及套餐信息。
- **购物车**：商品加入/移出购物车。
- **订单系统**：下单、微信支付、查看历史订单记录、再次下单。
- **催单与收货**：对长期未接的订单进行催单。
